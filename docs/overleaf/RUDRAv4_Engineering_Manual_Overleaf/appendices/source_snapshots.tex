\chapter{Implementation Source Index}

This manual does not vendor duplicate copies of implementation source files. The RUDRAv4 repository is the single source of truth for code, launch files, firmware, configuration, and scripts. This avoids a slow drift where the PDF contains one version of a node and the robot runs another.

The links below point to branch \texttt{\RepoBranch}. For a release PDF, update \cmd{\RepoBranch} in \filep{manual.sty} to the release tag or commit branch used to build the robot.

\begin{operatorbox}{Maintenance rule}
When implementation changes, update the source file in the repository first. Then update the manual only if the interface, behavior, operating procedure, theory, or safety envelope changed. Do not copy source into the Overleaf package.
\end{operatorbox}
