RUDRAv4 Startup

Field bringup guide for launching the NUC, Uno PS2 bridge, Teensy, and Sabertooth drive chain.

Normal Manual Drive Chain

PS2 controller
Paired receiver sends stick state to the Uno.
Arduino Uno
USB serial line: J,select,ry,rx,ly,lx.
NUC ROS2 bridge
Publishes raw values and sends drive lines to Teensy.
Teensy
USB serial input: D,left,right,enable. Responds with ACK,....
Sabertooth drivers
Packet serial from Teensy Serial1/TX1 at 9600 baud.

Before Battery Power

Put the wheels off the ground for the first powered test. Keep one hand ready to remove battery power. Close Arduino Serial Monitor before starting ROS, because it can hold the Uno or Teensy serial port.
  1. Confirm the PS2 receiver is paired and the Uno serial monitor previously showed changing J,... values.
  2. Confirm the Teensy onboard LED heartbeat is blinking twice per second.
  3. Confirm the Teensy serial monitor previously showed BOOT,RUDRA_TEENSY_SABERTOOTH_READY.
  4. Confirm no duplicate bridge is running: only one node should own the Teensy serial port.

Build and Source

cd /home/rudra/Projects/RUDRAv4
source /opt/ros/lyrical/setup.bash
colcon build --packages-select rudra_base_bridge
source install/setup.bash

Check Serial Devices

bash scripts/list_serial.sh

The current expected persistent ports are:

DevicePath
Uno/dev/serial/by-id/usb-Arduino__www.arduino.cc__0043_959313232323510182C0-if00
Teensy/dev/serial/by-id/usb-Teensyduino_USB_Serial_9210670-if00

Launch Manual PS2 Drive

cd /home/rudra/Projects/RUDRAv4
source /opt/ros/lyrical/setup.bash
source install/setup.bash
bash scripts/run_ps2_bridge.sh

Press SELECT on the PS2 controller to enable manual mode. Sticks at rest should keep left and right commands at zero.

Verify With Terminals

Open new SSH terminals, source the workspace, and watch these topics.

cd /home/rudra/Projects/RUDRAv4
source /opt/ros/lyrical/setup.bash
source install/setup.bash
ros2 topic echo /rudra/ps2_raw
ros2 topic echo /rudra/drive_cmd
ros2 topic echo /rudra/teensy_ack
Healthy manual drive looks like J,1,..., then nonzero D,left,right,1 when a stick moves, and matching ACK,left,right,1 from the Teensy.

Teensy Only Test

Use this when you want to test NUC to Teensy to Sabertooth without the Uno or PS2 controller.

cd /home/rudra/Projects/RUDRAv4
source /opt/ros/lyrical/setup.bash
source install/setup.bash
bash scripts/run_cmd_vel_bridge.sh
ros2 topic pub --rate 20 /cmd_vel geometry_msgs/msg/Twist "{linear: {x: 0.2}, angular: {z: 0.0}}"

Expected topics:

/rudra/drive_cmd  -> data: D,21,21,1
/rudra/teensy_ack -> data: ACK,21,21,1

Headless Checks

Without a monitor, use SSH from another machine on the same network.

ssh rudra@rudra.local

If mDNS is unavailable, use the NUC IP address instead.

ping rudra.local
ssh rudra@<NUC_IP_ADDRESS>

Once connected, these commands tell you whether RUDRA is launched:

source /opt/ros/lyrical/setup.bash
source /home/rudra/Projects/RUDRAv4/install/setup.bash
ros2 node list
ros2 topic list
ros2 topic echo --once /rudra/drive_cmd
ros2 topic echo --once /rudra/teensy_ack
SignalMeaning
Teensy LED blinkingNew Teensy firmware is running.
/ps2_uno_to_teensy in ros2 node listManual drive bridge is launched.
/rudra/ps2_raw changingUno and PS2 are reaching ROS.
/rudra/teensy_ack printingTeensy is receiving and replying to commands.

Launch on NUC Boot

Yes, RUDRA can launch automatically when the NUC boots. A systemd service is the cleanest approach.

Create the service

sudo nano /etc/systemd/system/rudra-ps2-bridge.service

Paste this service:

[Unit]
Description=RUDRAv4 PS2 to Teensy bridge
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=rudra
WorkingDirectory=/home/rudra/Projects/RUDRAv4
ExecStartPre=/bin/sleep 5
ExecStart=/bin/bash -lc 'source /opt/ros/lyrical/setup.bash && source /home/rudra/Projects/RUDRAv4/install/setup.bash && ros2 launch rudra_base_bridge ps2_to_teensy.launch.py config:=/home/rudra/Projects/RUDRAv4/src/rudra_base_bridge/config/rudra_v4_hardware.yaml'
Restart=on-failure
RestartSec=2

[Install]
WantedBy=multi-user.target

Enable and start

sudo systemctl daemon-reload
sudo systemctl enable rudra-ps2-bridge.service
sudo systemctl start rudra-ps2-bridge.service

Check status without a monitor

systemctl status rudra-ps2-bridge.service --no-pager
journalctl -u rudra-ps2-bridge.service -f

Stop or disable autostart

sudo systemctl stop rudra-ps2-bridge.service
sudo systemctl disable rudra-ps2-bridge.service

Shutdown

  1. Release the sticks.
  2. Press SELECT to disable manual mode if needed.
  3. Stop the ROS launch or service.
  4. Remove Sabertooth battery power.
  5. Shut down the NUC cleanly.
sudo shutdown now
