Remote SSH Bringup

Terminal-by-terminal commands for launching RUDRAv4 over SSH, watching drive topics, and opening RViz to see the LiDAR scan and obstacle guard.

Updated: 2026-06-06
Robot: RUDRAv4

SSH Setup

Use one SSH terminal per long-running process. If you want RViz through SSH, connect with trusted X forwarding from your laptop.

ssh -Y rudra@ROBOT_IP

If RViz feels slow over SSH, run RViz on the robot desktop instead, or use a remote desktop session. The ROS commands are the same once you are on the robot.

Keep the rover lifted or on blocks for first tests. Do not run ps2_uno_to_teensy and cmd_vel_to_teensy at the same time because both own the Teensy serial port.

Every Terminal

Run this in each SSH terminal before launching or echoing ROS topics.

cd ~/Projects/RUDRAv4
source /opt/ros/lyrical/setup.bash
source ~/ros2_ws/install/setup.bash
source ~/Projects/RUDRAv4/install/setup.bash

If you just changed code, rebuild once before launching:

cd ~/Projects/RUDRAv4
source /opt/ros/lyrical/setup.bash
source ~/ros2_ws/install/setup.bash
colcon build --packages-select rudra_base_bridge
source install/setup.bash

Terminal 1: Serial Check

Confirm the NUC sees the Uno, Teensy, and LiDAR USB adapter.

cd ~/Projects/RUDRAv4
bash scripts/list_serial.sh

Terminal 2: Start LiDAR

This starts the YDLIDAR G2B driver and publishes /scan.

cd ~/Projects/RUDRAv4
bash scripts/run_lidar.sh /dev/ttyUSB0

Wait up to about 15 seconds for the first scan. Startup checksum warnings can appear, but the important line is that /scan begins publishing.

To stop the driver and request motor stop:

cd ~/Projects/RUDRAv4
bash scripts/stop_lidar.sh
If the LiDAR keeps spinning after no LiDAR ROS process is running, it is being powered directly by USB. Unplug the LiDAR USB cable or switch off its USB power/hub port.

Terminal 3: Start Manual Robot Bridge

This reads the PS2 controller from the Uno and sends drive commands to the Teensy.

cd ~/Projects/RUDRAv4
bash scripts/run_ps2_bridge.sh

Press SELECT on the PS2 controller to enable manual drive. Press X while stopped to toggle obstacle avoidance after the Uno has been reflashed with the updated firmware.

For future autonomy, do not run this manual bridge at the same time as the /cmd_vel bridge. Use one owner of the Teensy serial port at a time.

Terminal 4: Watch Topics

Use these to see the raw controller packet, motor command, Teensy acknowledgements, scan data, and obstacle guard decisions.

cd ~/Projects/RUDRAv4
source /opt/ros/lyrical/setup.bash
source ~/ros2_ws/install/setup.bash
source ~/Projects/RUDRAv4/install/setup.bash

ros2 topic echo /rudra/ps2_raw
ros2 topic echo /rudra/drive_cmd
ros2 topic echo /rudra/teensy_ack
ros2 topic echo /rudra/obstacle_guard
ros2 topic echo --once /scan
TopicMeaning
/rudra/ps2_rawRaw Uno packet. Updated firmware should show J,select,ry,rx,ly,lx,x.
/rudra/drive_cmdFinal command sent to Teensy, after obstacle guard filtering.
/rudra/teensy_ackTeensy acknowledgement and motor-side status.
/rudra/obstacle_guardText status such as clear, slowing, blocked, or disabled.
/scanRaw LiDAR LaserScan used by the guard and future SLAM/Nav.

Terminal 5: Open RViz

Use this from an SSH session with X forwarding, or from a terminal on the robot desktop.

cd ~/Projects/RUDRAv4
bash scripts/view_lidar.sh

RViz opens with fixed frame laser. The green points are the raw /scan. The marker sphere is the closest obstacle that the guard is using in the front cone.

RViz ColorMeaning
Green pointsRaw LiDAR scan from /scan.
Green markerGuard sees front cone clear.
Orange markerGuard is slowing forward drive.
Red markerGuard is blocking forward drive.
Gray markerGuard is disabled, but scan can still publish for SLAM/Nav.

Optional: Test With /cmd_vel

Use this path later for keyboard teleop, SLAM, or Nav2. Stop the PS2 bridge first so the Teensy serial port is free.

cd ~/Projects/RUDRAv4
bash scripts/run_cmd_vel_bridge.sh

In another terminal, send a small forward command with the wheels lifted:

ros2 topic pub --rate 5 /cmd_vel geometry_msgs/msg/Twist \
  "{linear: {x: 0.15}, angular: {z: 0.0}}"

Stop publishing with Ctrl-C. The bridge sends stop commands when /cmd_vel times out.

Quick Shutdown

cd ~/Projects/RUDRAv4
bash scripts/stop_lidar.sh

Press Ctrl-C in the PS2 bridge, RViz, and topic-monitor terminals. If the LiDAR still spins after software shutdown, remove USB power from the LiDAR.
